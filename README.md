# Бот «Статус врача»

Отвечает на фамилию врача из сводки статусом: логин, прогресс по курсам,
дни без активности, контекст по договору ReStaff.

## Шаг 1. Получить токен у @BotFather (2 минуты)

1. В Telegram открой [@BotFather](https://t.me/BotFather).
2. Отправь `/newbot`.
3. Придумай имя (например, «Статус врача») — любое, отображается пользователям.
4. Придумай username, обязательно заканчивается на `bot`, например
   `doctor_status_test_bot`. Это и есть твоя финальная ссылка:
   `https://t.me/doctor_status_test_bot`.
5. BotFather пришлёт токен вида `123456789:AAExampleTokenHere`. Сохрани его.

## Шаг 2. Задеплоить код (вариант — Railway, бесплатно, ~10 минут)

1. Зайди на [railway.app](https://railway.app), зарегистрируйся (можно через GitHub).
2. Залей эти два файла (`main.py`, `requirements.txt`) в новый репозиторий на
   GitHub — самый простой способ: создать репозиторий на github.com прямо в
   браузере, кнопка "Add file → Upload files", перетащить оба файла, Commit.
3. В Railway: **New Project → Deploy from GitHub repo** → выбери репозиторий.
4. В настройках проекта (Variables) добавь переменную:
   - `BOT_TOKEN` = токен из Шага 1.
5. Railway сам определит, что это Python-проект, установит зависимости из
   `requirements.txt` и запустит `python main.py`.
6. Дождись в логах строки о старте polling — бот online.

Готово: пиши боту по адресу из Шага 1 (`@doctor_status_test_bot` или как ты
его назвала) — он должен отвечать.

### Альтернатива без GitHub — Replit

1. На [replit.com](https://replit.com) создай Python-repl.
2. Вставь содержимое `main.py` и `requirements.txt`.
3. В разделе **Secrets** (значок замка) добавь `BOT_TOKEN` со значением токена.
4. Нажми Run. Чтобы бот не засыпал — подключи Replit "Always On" (платная
   опция) или внешний пингер типа UptimeRobot на URL репла, если Replit
   даёт веб-адрес; для чисто polling-бота типа этого можно просто держать
   вкладку с запущенным Run открытой на время проверки.

### Локальный запуск (для проверки у себя, без хостинга)

```bash
pip install -r requirements.txt
export BOT_TOKEN="123456789:AAExampleTokenHere"   # Windows: set BOT_TOKEN=...
python main.py
```

Пока скрипт запущен — бот отвечает. Закрыл терминал — бот офлайн. Для
демонстрации проверяющему нужен вариант с постоянным хостингом (Railway/Replit
Always On) либо запуск на своём компьютере на момент проверки.

## Проверка

Напиши боту `/start` — увидишь список врачей, по которым есть данные.
Затем отправь любую фамилию, например `Иванова`, `Петров`, `Козлов`.

## База данных

Зашита в `main.py` (словарь `DOCTORS`) на основе сводки за 09.07 и таблицы
доп. контекста. В конце файла есть комментарий, как заменить статичный
словарь на живое подключение к Google Sheets, чтобы сводка обновлялась
каждый день без переразвёртывания кода.
